HTML

  • HTML, or Hypertext Markup Language is hte basic building block of the web. It defines the meaning and structure of web content.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements are the whole thing while tags are the content inbetween the brackets.
  • Head data is for the browser while body data is displayed on the webpage.
  • Attributes extend the HTLM element, changing its behavior or provides metadata.
  • Script Tags are an HTML element that is used to store, among other things, JavaScript code.

CSS

  • CSS stands for Cascading Style Sheets.
  • CSS is a stylesheet language used to describe the presentation of a document in HTML or XML.
  • CSS can be embedded in the HTML file or seperate from it.
  • CSS will by default apply to all of the HTML structure.
  • Responsive Design is a web design approach to make web pages that render well on all screen sizes and resolutions while ensuring good usability in order to design a multi-device web.
  • CSS Resets are a way of making the styling across browsers consistent with each other. Think the normalize file you use.
  • Creating classes allows us to differentiate which elements are being affected by the CSS file.
  • Pseudo-Classes in CSS is a keyword added to a selector that specifies a special state of the selected element(s). Think :hover, :checked, :visited.
  • Psuedo-Elements in CSS is a keyword added to a a selector that lets you style a specific part of the selected element(s). This is symbolized with two colons (::).
  • CSS Variables are custom properties set to a variable and declared in the :root selector and are then used throughout the CSS using the declared variable.
  • Color is the CSS property that sets the color value.
  • Font is the CSS property that sets the font value.
  • Typography is the technique/art of arranging type on a page and includes the font family, style, size, weight, and spacing.
  • Wireframes are a page schematic or screen blueprint that represents the skeletal framework of a website.
  • Display is the CSS proprerty that determines whether an element is treated as inline or block and the layout used for its children, such as flow layout, grid, or flex.
  • Flexbox is a module as a method that can offer space distribution between items in an interface and powerful alignment.
  • Position is the CSS property that sets how an element is positioned in the document and may include top, right, bottom, and left properties.
  • Selectors are CSS rules that describe what elements in a document the rule will match. Matching elements get that style applied.
  • Advanced CSS Selectors have various ways of combining elements/selectors to style specific elements. Includes things such as the Descendent Combinator, Child Combinator, Adjacent Sibling, Combinator, and General Sibling Combinators.
  • The Box Model is that everything in CSS has a box around it and includes content, padding, border, and margin.
  • Advanced Box Styling is using the CSS box model and basic CSS layouts, one can make content with unique backgrounds, circles, a cloud, blockquote, etc.
  • Media Queries allow you to apply CSS styles depending on the devices general type, screen resolution, or browser viewpage width.
  • Transitions in CSS are the properties that define the transition of two states of an element.

The Command Line and Git

  • Basic Built-In Terminal Commands are basic commands already found in the command line such as cd, touch, curl, and mkdir.
  • curl is used in command lines or scripts to transfer data.
  • git status: checks what branch we are currently on.
  • git check -b branch-name: creates a new branch and switches to it.
  • git add -A: adds the files in the working directory.
  • git commit -m "note": commits to our repository.
  • git pull origin main: updates our current branch
  • git push origin "feature branch": adds our changes to the repository.
  • git stash is for when you want to record the current state of the working directory and the index, but want to go back to a clean working directory.

GitHub

  • GitHub is a code hosting platform for version control and collaboration.
  • Version Control is the process of updating a project while holding on to prior history in case of need to revert.

JavaScript

  • Enter your JavaScript notes here
  • A variable is a named container that allows us to store data in our code. It is hoisted and then supplied a value, otherwise it is undefined.
  • Control flow is the order in which a computer executes code in a script.
  • Primitives are data types that are not an object and have no methods or properties. Primitive data types include strings, numbers, bigint, boolean, undefined, symbol, and null.
  • Logical and Comparison Operators are symbols for logic and arithmetic and comparison.
  • Conditional Statements are statements that result in an output if the condition is true or doe else if it is false.
  • Arrays are objects that enable storing a collection of multiple items under a string variable name, and has numbers for performing common array operations.
  • Array Methods are actions that are performed on arrays. These are functions that are stored as object properties.
  • String Methods are actions performed on strings.
  • Objects are one of JavaScript's data types. It is used to store various keyed collections and more complex entities.
  • Object Methods are a property of an object that is a function.
  • Iteration loops offer a quick and easy way of doing things repeatedly.
  • Functions are a fundamental building block in JavaScript. It is a procedure. A set of statements that performs a task or calculates a value, but also takes in an input and generates an output.
  • Scoping is the range at which a variable can register in the code which can be global or local. There is other scope terms such as module or block.
  • "This" keyword is a reference to an object that the keyword is use and can be either function, class, or global.
  • The Window Object is the window interface which represents a window contain a DOM document.
  • Document Object Model (DOM) connects the web pages to scripts by representing the structure of a document. The DOM represents a document with a logical tree, where the branch is a node that can have events connected to it.
  • Querying the DOM is the document method, such as querySelector which returns a static NodeList representing a list of the documents elements that match the specified group of selectors.
  • The Style Property returns the inline style of an element.
  • Setting Element Attributes involves setting a the value of an attribute on the specified document. If the attribute exists, then it is updated. get/remove.Attribute().
  • Creating DOM Elements requires a method that creates an HTML element by specific tag name. object.createElement('tag name').
  • Appending Elements requires a method, appendChild() which adds an element to a specified parent node.
  • Intervals and Timers requires a method, setInterval() which repeatedly calls a function or executes a code snippet, with a fixed time delay between each call. This can be removed with clearInterval().
  • Event Listeners is a user interaction with the keyboard. Each event describes a single interaction between the user and a key.
  • Preventing Default requires a method, preventDefault in which the event interface tells the user agent that if the event does not get explicitly handled, its default action should not be taken as it normally would be.
  • Stopping Propagation requires a method of the event interface and prevents the propagation of a current event. stopPropagation().
  • Storage is located in the storage interface which provides access to a particular domain session, or local storage.
  • Data Attributes allow us to store extra information on standard semantic HTML elements.
  • ES6 stands for EcmaScript 6th Edition which the standard that JavaScript is based off. This edition includes let/const keywords, arrow functions, template literals, and more.
  • Let is a declaration that is block-scoped as a local variable.
  • Const is a declaration that creates block-scoped constants. It cannot be changed through reassignment.
  • Arrow Function Expressions are compact alternatives to a traditional function expression. It should not be used as methods. It cannot be used as a constructor and cannot be created as a gen function.
  • Template Literals use the backtick key to concatenate text use ${} to substitute values instead of +.
  • Asynchronous JavaScript is the ability that allows you to run code while waiting on other processes to complete.
  • Callbacks are functions passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action.
  • Promises are objects that represent eventual completion, or failure of an asynchronous operation and its resulting value.

Server Side APIs

  • Server Side APIs are application programming interfaces for either a web server or a web browser. Server-side is a programmatic interface consisting of one or more publicly exposed endpoints to a define requested response message system, typically expressed in JSON or XML
  • Client Server Model is a distributed application structure that partitions tasks or workloads between the providers of a resource or service, called servers.
  • JSON is objects containing methods for parsing JavaScript Object Notation (JSON) and converting values to JSON. JSON is static like the Math object.
  • URL is the URL interface used to parse, construct, normalize, and encode URLs. It works by providing properties which allow you to easily read and modify the components of a URL.
  • The HTTP GET is a method requests a representation of the specific resource. Requests using Get should only be used to request data (they shouldn't include data).
  • The HTTP Status Code indicates whether a specific HTTP request has been successfully completed responses are grouped under the classes informational, successful, redirection, client error, and server error.
  • Fetch API provides an interface for fetching resources. More powerful and flexible than XML HTTP Requests.
  • Location API is a location interface that represents the location (URL) of the object it is linked.
  • Node.js is software that executes JavaScript outside of the browser so developers can write command line tools and allow server-side scripting.
  • Process is one of the two main objects in node (the other is global). Process.argv returns an array of what was put into the command line when node.js process was launched. The first index will be the path to node, the second index will return the file passed. Additional arguments make up the rest of the array.
  • fs is a module in node that allows the code to work on the computer file system. This includes reading, creating, updating, deleting, and renaming.
  • Modularization is seperating program's functions into independent pieces or buidling blocks, each containing all the parts needed to execute a single aspect of functionality.
  • require() is used to import modules, JSON, and local files.
  • module.exports allows a function to be saved globally for other .js files in the same package.
  • npm stands for Node Package Manager which is a software library that comes with node.js. You can download a variety of npm packages to do different things in JavaScript.
  • package.JSON is a file that lists the packages your project depends on.
  • npm init creates the package.json file.
  • node.modules is the folder where the installed npm packages are stored.
  • inquirer is a package that makes the command line interface interactive.
  • Array.prototype.map() is a method that creates a new array populated with the results of calling a provided function on every element in the calling array.
  • Array.prototype.reduce() is a method that takes a callback function and reduces it to a single value.
  • Spread Syntax allows you to iterate an array.

Object Oriented Programming

  • Objects are datatypes that store various keyed collections and more complex entities.
  • Prototype is the base set of properties/mehtods embedded on any object.
  • Inheritance is where everything has a relationship to something called a prototype that eventually results in null.
  • This refers to the parent object that it is evoked from.
  • Polymorphism is one block of code and changing one version based on the inputs we are giving it.
  • Method Overloading is when a class has multiple methods by the same name but different parameters.
  • Promise is when an object represents the eventual completion (or failure) of an asynchronous operation and its resting value.
  • Callback Function is passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action.
  • Constructor method is a special method of a class for creating the intiializing an object instance of that class.
  • Function Constructors are patterns for creating empty objects and setting to a variable. This pattern is an alternative to factory functions.
  • Test Driven Development (TDD) is a software development process relying on software requirements being coverted to test cases before software is fully developed and tracks all software dev by repeatedly testing software against all test cases.
  • Jest is a JavaScript testing framework designed to ensure correctness of only JavaScript codebase. Sends results quickly.
  • Mock Functions allow you to test the links between code by erasing the actual implementation of a function, capturing calls to the function (and the parameters passed in those calls), capturing instances of constructor functions when instantiated with new, and allowing test-time configuration of return values.
  • Mocking Modules creates a hypothetical success (or failure) to test the rest of the code that has been written.
  • Classes are templates for creating objects. They encapsulate data with code to work on that data. JS classes are built on prototype.
  • SubClasses are classes which inherit both the properties of that class without editing the class itself.
